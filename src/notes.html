# FCC-WIKI-SEARCH ## A wikipedia search engine challenge on freecodecamp.com I decided to create this application using Angular, release 4.x, the decision to do so was not done lightly as it has involved a significant investment of time and money to aquire this knowledge. I've been fighting with new concepts every inch of this mile. Most probably have no trouble with but this was not for the faint of heart as I've just begun to scratch the surface. I am confident that the finished product will meet my expectations. The skeleton of this project is based loosely off Patrick Stapleton's angular2-webpack-starter RxJS example, and the NG-BOOK2 YouTube Search app. I did not copy this code, it is my own work. ### The code breakdown 1. `SearchResult` object to hold data from each result 2. `SearchService` manage the API requests and convert results to streams of type SearchResult[] 3. `SearchBoxComponent` calls out to the SearchService while query is typed 4. `SearchResultComponent` renders a specific SearchResult 5. `SearchComponent` encapsulates app and renders list of results 6. `SearchInjectables` environmental constants to be injected ## 1. @SearchResult - /app/src/search/search.result.model.ts Object that defines the SearchResult object type is built to match the Wikipedia API. Will need to store the following strings: 1. id 2. linkUrl (`https://en.wikipedia.org/?curid=+id`) 3. title 4. description Will call Wikipedia API with URL: `https://en.wikipedia.org/w/api.php?action=opensearch&format=json&search=` ```ng generate component search``` ## 2. @SearchService - /app/src/search/search.service.ts Location of constants that will be injected ## 3. @SearchBoxComponent: ## 4. @SearchResultComponent: ## 5. @SearchComponent ## 6. @SearchInjectables - /app/src/search/search.injectables.ts Environmental constants to be injected
